HTML

  • The head element contains information about the webpage.
  • meta tags give information about the webpage.
  • title sets the title for the page and is used in SEO.
  • link tags attach other documents to the webpage such as external css files, web fonts and icon libraries.
  • script tags contain client-side scripts.
  • The body element represents the visible content shown to the user.
  • h1 - h6 are heading tags, they make text more prounounced and are used semantically, only one h1 heading per page.
  • img displays an image.
  • ul (unordered list) renders a bullet pointed list.
  • ol (ordered list) is like ul but numbered.
  • li has to be nested inside ul or ol and each one produces a list item.

CSS

  • CSS can be written as inline, internal stylesheets or most commonly external stylesheets
  • External stylesheets must be linked in the head using a link tag and two attributes, rel with the value of stylesheet and href which will include the path to the file (use ./ to use a relative path from the current html file)
  • We then use selctors, properties and values like this: selector {propery: value;}
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • * selects all visible elements, we usually start off a css file with * {margin: 0; padding: 0;}
  • More than one selector can be targeted at a time, if you want to target to headings: h1, h2 {padding: 20px;}
  • Box shadows create a shadow around an element, it uses the values of x, y, blur, spread and color.
  • Text-align decides what alignment text will have, left, right, center or justified.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stage all files that have been modified, ready to be commited
  • git commit -m "commit comment shere": to commit files to git
  • git pull origin main: make sure you have the most recent version of the main branch
  • git push origin feature/branch-name: push changes to main branch
  • git pull origin main frequently to make sure have most current version

Git Flow steps

  • Navigate to folder
  • git status - to check where you are?
  • If not on the main branch - git checkout main
  • Pull remote branch into local branch, to make sure up to date - git pull origin main
  • Create new branch - git checkout -b feature/branch-name
  • Once you are finished on branch:
  • Add all changes to the staging area - git add -A
  • Commit the changes - git commit -m "suitable commit message"
  • Check local branch is up to date - git pull origin main
  • When ready push local changes to remote branch - git push origin feature/branch-name
  • When you have completed everything you need to do on this branch:
  • Go to GitHub > Pull Requests > Create a new pull request
  • Check you are merging the correct branch to the main branch...
  • Give title and description on what's been done...
  • Confirm pull request
  • Confirm merge
  • Close any issues if completed
  • Go back to Git Bash and update main branch again - git pull origin main

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Create a .js file to write JavaScript code in.
  • Link a JavaScript file using script tag with a src attribute to point to where the file is.
  • Variables store data, declare a variable by typing var nameOfVar.
  • Assign a value with = "value" for a string.
  • console.log() allows me to print something to the console to see what's going on with my JS code.
  • Arrays are variables that can store more than one value and are accessed by an index value (0 being the first entry).
  • To access a value of an array call on it and add the index number in square brackets: myArray[1]
  • An array can store all different data types in one array.
  • For loops allow us to iterate over an array or other code, so we don't have to keep repeating code.
  • Functions contain code that can be called upon whenever we want and however many times we want.
