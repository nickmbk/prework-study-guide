Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • meta tags give information about the webpage.
  • title sets the title for the page and is used in SEO.
  • link tags attach other documents to the webpage such as external css files, web fonts and icon libraries.
  • script tags contain client-side scripts.
  • The body element represents the visible content shown to the user.
  • h1 - h6 are heading tags, they make text more prounounced and are used semantically, only one h1 heading per page.
  • img displays an image.
  • ul (unordered list) renders a bullet pointed list.
  • ol (ordered list) is like ul but numbered.
  • li has to be nested inside ul or ol and each one produces a list item.

CSS

  • CSS can be written as inline, internal stylesheets or most commonly external stylesheets
  • External stylesheets must be linked in the head using a link tag and two attributes, rel with the value of stylesheet and href which will include the path to the file (use ./ to use a relative path from the current html file)
  • We then use selctors, properties and values like this: selector {propery: value;}
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • * selects all visible elements, we usually start off a css file with * {margin: 0; padding: 0;}
  • More than one selector can be targeted at a time, if you want to target to headings: h1, h2 {padding: 20px;}
  • Box shadows create a shadow around an element, it uses the values of x, y, blur, spread and color.
  • Text-align decides what alignment text will have, left, right, center or justified.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stage all files that have been modified, ready to be commited
  • git commit -m "commit comment shere": to commit files to git
  • git pull origin main: make sure you have the most recent version of the main branch
  • git push origin feature/branch-name: push changes to main branch
